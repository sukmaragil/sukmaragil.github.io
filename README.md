# Sukma Ragil — personal site

A static personal site + blog built with [Jekyll](https://jekyllrb.com), hosted free on **GitHub Pages**. GitHub builds the site automatically every time you push — no manual export step.

---

## Folder structure

```
_config.yml              Site settings (title, links, baseurl)
index.html               Home page
work.html                Experience page  (content from _data/jobs.yml)
projects.html            Projects page    (content from _data/projects.yml)
blog.html                Blog index (auto-lists everything in _posts/)
_layouts/
  default.html           Shared header / nav / footer
  post.html              Layout for a single blog post
_data/
  jobs.yml               Your work history — edit here
  projects.yml           Your projects — edit here
_posts/                  One Markdown file per blog post
assets/css/style.css     All styling
```

---

## ✍️ How to publish a new blog post

1. Create a file in `_posts/` named `YEAR-MONTH-DAY-title.md`, e.g.
   `_posts/2026-06-20-my-new-post.md`
2. Start it with this front matter, then write the body in Markdown:

   ```markdown
   ---
   layout: post
   title: "My New Post"
   date: 2026-06-20
   read: "5 min"
   excerpt: "One or two sentences shown on the blog index."
   ---

   Write your post here. Plain paragraphs, **bold**, _italic_,
   `code`, lists, ## headings, > quotes — all standard Markdown.
   ```
3. Commit and push. The post appears at `/blog/my-new-post/` and on the blog index automatically (newest first).

That's the whole workflow — no rebuild on your machine required.

## 🧰 How to update work history / projects

- **Work:** edit `_data/jobs.yml` — copy a block to add a role.
- **Projects:** edit `_data/projects.yml` — copy a block to add a project.
- **Home intro & focus areas:** edit `index.html`.
- **Name, social links, email:** edit `_config.yml`.

Push the change and the live site updates.

## 🚀 Deploying to GitHub Pages

1. Create a GitHub repository and put **the contents of this folder** at the repo root (the `_config.yml`, `index.html`, `_posts/`, etc. should be at the top level — not inside a `jekyll-site/` subfolder).
2. **Repo → Settings → Pages → Build and deployment → Source: _Deploy from a branch_ → branch `main`, folder `/ (root)`.** Save.
3. Wait ~1 minute. Your site is live.

### Pick the right `baseurl` in `_config.yml`

- Repo named **`<username>.github.io`** → site is at `https://<username>.github.io/`
  Leave `baseurl: ""`.
- Any **other repo name** (e.g. `portfolio`) → site is at `https://<username>.github.io/portfolio/`
  Set `baseurl: "/portfolio"` so links and CSS resolve correctly.

## 💻 Previewing locally (optional)

You don't need this to publish, but to see changes before pushing:

```bash
gem install bundler jekyll
bundle install
bundle exec jekyll serve
# open http://localhost:4000
```

(`preview.html` in this folder is only a rough static mock-up of the look — the real pages are generated by Jekyll from the files above.)
